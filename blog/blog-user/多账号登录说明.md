# 多账号登录功能说明

## 功能概述
现在支持在不同浏览器标签页中同时登录不同的账号。每个标签页的登录状态是独立的，互不影响。

## 实现原理
- 使用 `sessionStorage` 替代 `localStorage` 存储用户认证信息
- `sessionStorage` 的特点是每个标签页拥有独立的存储空间
- 关闭标签页后，该标签页的登录状态会自动清除

## 使用方式
1. 在第一个标签页登录账号A
2. 打开新标签页（Ctrl+T 或 Cmd+T）
3. 在新标签页中可以登录账号B
4. 两个标签页可以同时使用，互不干扰

## 重要提示
**首次使用前请清理浏览器缓存：**
1. 按 F12 打开开发者工具
2. 在 Application（应用）标签页中
3. 找到 Local Storage 和 Session Storage
4. 清除所有相关数据
5. 刷新页面

或者直接使用浏览器的"清除浏览数据"功能（Ctrl+Shift+Delete）

## 注意事项
1. **登录状态仅在当前标签页有效**
   - 关闭标签页后需要重新登录
   - 刷新页面不会丢失登录状态

2. **数据隔离**
   - 每个标签页的用户数据（点赞、收藏、浏览历史等）是独立的
   - 不同标签页之间的操作不会相互影响

3. **应用配置共享**
   - 筛选条件、排序方式等应用配置在所有标签页间共享
   - 自定义标签、自定义分类等配置也是共享的

## 故障排查
如果遇到"一个标签页退出登录，另一个标签页也退出"的问题：

1. **清理浏览器缓存**
   - 清除所有 localStorage 和 sessionStorage 数据
   - 关闭所有标签页
   - 重新打开浏览器

2. **检查浏览器控制台**
   - 按 F12 打开开发者工具
   - 查看 Console 标签页是否有错误信息
   - 查看 Application > Storage 确认使用的是 sessionStorage

3. **确保代码已更新**
   - 确认所有文件都已保存
   - 重新启动开发服务器（npm run dev）
   - 强制刷新页面（Ctrl+Shift+R 或 Cmd+Shift+R）

## 技术细节
- 认证信息（token、userInfo）：使用 `sessionStorage`（标签页独立）
- 用户数据（点赞、收藏等）：使用 `sessionStorage`（标签页独立）
- 应用配置（筛选、排序等）：使用 `localStorage`（全局共享）
- Pinia store 持久化：使用 `sessionStorage`（标签页独立）

## 迁移说明
如果您之前已经登录过，首次使用新版本时需要：
1. 清理浏览器缓存
2. 重新登录
3. 旧的 localStorage 数据会在应用启动时自动清理
