### 2. 修复多账号登录功能 ✅

**文件**: `blog-user/src/views/ArticleDetail.vue`

**问题**:
- 之前使用 PowerShell 批量替换 localStorage 调用时可能导致编码问题
- 需要手动将所有 localStorage 调用替换为使用 `@/utils/storage` 工具类

**修复内容**:
1. 导入了 storage 工具类：`import { getItem, setItem } from '@/utils/storage'`
2. 替换了所有 localStorage 调用：
   - `localStorage.getItem()` → `getItem()`
   - `localStorage.setItem()` → `setItem()`

**替换的位置**:
- `getPurchasedArticles()` 函数
- `savePurchasedArticle()` 函数
- `loadArticle()` 函数中的浏览历史
- `handleLike()` 函数中的点赞列表
- `handleCollect()` 函数中的收藏列表
- `handlePurchase()` 函数中的已购文章列表
- `submitComment()` 函数中的 token 获取

**验证**:
- 使用 grepSearch 确认文件中已无 localStorage 调用
- 使用 getDiagnostics 确认文件无语法错误

## 存储策略说明

根据 `blog-user/src/utils/storage.js` 的设计：

### sessionStorage（标签页独立）
- **用户认证信息**: `token`, `userInfo`
- **用户数据**: `likedArticles`, `collectedArticles`, `browseHistory`
- **目的**: 实现多标签页独立登录，不同标签页可以登录不同账号

### localStorage（全局共享）
- **购买记录**: `purchasedArticles`
- **应用配置**: `filterCategory`, `sortBy`, `customTags`, `customCategories`
- **目的**: 购买是永久性的，应该在所有标签页共享；配置也应该全局共享
