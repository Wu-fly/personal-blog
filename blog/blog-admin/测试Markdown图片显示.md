# 测试 Markdown 图片显示功能

## 测试步骤

### 1. 创建测试文章

在用户前端创建一篇包含图片的文章：

1. 登录用户账号
2. 进入文章编辑页面
3. 输入标题和内容
4. 在内容中粘贴图片（会自动生成 Markdown 格式：`![图片](url)`）
5. 提交文章等待审核

### 2. 管理员审核

1. 登录管理员账号（端口 5175）
2. 进入"文章审核"页面
3. 找到刚才提交的文章
4. 点击"预览"按钮

### 3. 验证结果

预览对话框应该显示：

✅ **封面图片**（如果有）
- 显示在文章标题下方
- 居中显示
- 有圆角和阴影

✅ **文章内容中的图片**
- Markdown 格式 `![图片](url)` 应该转换为 `<img>` 标签
- 图片正常显示，不是 URL 文本
- 图片自适应宽度
- 有圆角和阴影

✅ **其他 Markdown 格式**
- 标题正确显示（# ## ###）
- 粗体和斜体正确显示
- 链接可以点击
- 代码块有背景色

## 示例文章内容

```markdown
# 测试文章标题

**粗体文字** 和 *斜体文字*
```

## 预期效果

在管理员预览中，上面的 Markdown 内容应该渲染为：

- 标题有不同的字体大小
- 图片正常显示（不是 URL 文本）
- 粗体和斜体有样式
- 链接可以点击
- 代码有背景色

## 常见问题

### Q: 图片还是显示为 URL 文本？

A: 检查以下几点：
1. 确认文章内容是 Markdown 格式（包含 `![图片](url)`）
2. 检查浏览器控制台是否有错误
3. 确认图片 URL 是否可访问
4. 刷新页面重试

### Q: 图片显示但样式不对？

A: 检查 CSS 样式是否正确加载：
1. 打开浏览器开发者工具
2. 检查 `.article-content :deep(img)` 样式是否生效
3. 确认没有其他样式覆盖

### Q: 封面图片不显示？

A: 检查：
1. 文章是否有 `coverImage` 字段
2. `coverImage` URL 是否有效
3. 检查 `v-if="currentArticle.coverImage"` 条件是否满足

## 技术说明

### Markdown 转换流程

1. 前端编辑器保存文章时，图片以 Markdown 格式存储：`![图片](url)`
2. 后端数据库存储原始 Markdown 文本
3. 管理员预览时，前端检测内容格式：
   - 如果包含 HTML 标签（`<img`, `<h1>` 等），直接使用
   - 如果是纯文本/Markdown，调用 `convertMarkdownToHtml()` 转换
4. 转换后的 HTML 通过 `v-html` 渲染到页面

### 支持的 Markdown 语法

- `![alt](url)` → `<img src="url" alt="alt" />`
- `[text](url)` → `<a href="url">text</a>`
- `# 标题` → `<h1>标题</h1>`
- `**粗体**` → `<strong>粗体</strong>`
- `*斜体*` → `<em>斜体</em>`
- `` `代码` `` → `<code>代码</code>`
- ` ```代码块``` ` → `<pre><code>代码块</code></pre>`
- `> 引用` → `<blockquote>引用</blockquote>`
- `* 列表` → `<ul><li>列表</li></ul>`

### 为什么不在后端转换？

1. **灵活性**: 前端可以根据不同场景使用不同的渲染方式
2. **性能**: 减轻后端压力，转换在客户端进行
3. **原始数据**: 数据库保存原始 Markdown，便于后续编辑
4. **兼容性**: 同时支持 HTML 和 Markdown 两种格式
